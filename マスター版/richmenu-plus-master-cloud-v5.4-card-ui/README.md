# リッチメニュープラス マスター版（クラウド保存対応）

自分用に使うマスター版です。  
ログイン画面・パスワード確認は使わず、起動後すぐに編集画面を開きます。

## この版の位置づけ

- マスター版：自分用。パスワードなし。クライアント選択・トークン切替あり。
- クラウド保存対応版：Google Apps Script / Googleスプレッドシート / Google Drive に、画像込みプロジェクトを保存・読み込みできます。
- 通常版：クライアント配布用。パスワード制限を入れる予定。

## 主な機能

- リッチメニュー画像の読み込み
- タップエリア作成
- URI / メッセージ / Postback / リッチメニュー切替アクション設定
- LINE APIへのアップロード
- アップロード後の自動デフォルト設定
- デフォルトリッチメニュー設定
- デフォルト解除
- 現在のデフォルトrichMenuId確認
- ユーザー個別リンク中richMenuId確認
- 登録ユーザーへのリンク / リンク解除
- クライアント別チャネルアクセストークン保存
- お手軽保存（ブラウザ内保存のみ）
- バックアップ保存（JSON＋PNG）
- クラウド保存（画像込みプロジェクトをGoogle側に保存）
- クラウドから読み込み
- Apps Script用コードの表示・コピー

## クラウド保存で保存されるもの

クラウド保存では、画像込みのプロジェクトJSONをGoogle Driveに保存し、一覧情報をGoogleスプレッドシートに記録します。

スプレッドシートに記録されるもの：

```txt
クラウドID
クライアント名
保存名
更新日時
DriveファイルID
トークン保存
メニュー数
メモ
```

以前の版で英語見出しが作成されている場合も、接続テストや保存時に日本語見出しへ整えます。既存データは同じ列順で読み取るため、そのまま使えます。

Google Driveに保存されるもの：

```txt
画像込みプロジェクトJSON
- メニュー設定
- 背景画像データ
- タップエリア
- アクション設定
- チャネルアクセストークン
- クライアント一覧
- richMenuId
- 最終保存情報
```

## クラウド保存の使い方

### 最初に1回だけ行うApps Script設定

1. アプリを開く
2. 右側の「保存管理（クライアント管理）を開く」を押す
3. 「クラウド保存（Googleスプレッドシート）」を開く
4. 「Apps Script用コードを開く」を押す
5. 「コードをコピー」を押す
6. [Google Apps Script](https://script.google.com/) を開く
7. 「新しいプロジェクト」を押す
8. プロジェクト名を「リッチメニュープラス_クラウド保存」など、分かりやすい名前に変更する
9. 最初から入っている `function myFunction()` などのコードをすべて削除する
10. コピーしたApps Script用コードを貼り付ける
11. 保存アイコン、または `Ctrl + S` で保存する
12. 右上の「デプロイ」→「新しいデプロイ」を押す
13. 種類の選択で歯車アイコン、または「種類を選択」から「ウェブアプリ」を選ぶ
14. 説明欄に「リッチメニュープラス クラウド保存」などを入力する
15. 「次のユーザーとして実行」は「自分」を選ぶ
16. 「アクセスできるユーザー」は「全員」を選ぶ
17. 「デプロイ」を押す
18. 初回はGoogleの承認画面が出るため、「アクセスを承認」を押す
19. 「このアプリはGoogleで確認されていません」と表示された場合は、「詳細」→「プロジェクト名に移動」を押し、内容を確認して許可する
20. 承認後に表示される「ウェブアプリURL」をコピーする
21. リッチメニュープラスに戻り、「Apps Script URL」欄に貼り付ける
22. 「接続テスト」を押す
23. 「接続OK」と表示されたら設定完了

接続テストが成功すると、自分のGoogleアカウント内に以下が作成されます。

```txt
Google Driveフォルダ：RichMenuPlus_CloudData
Googleスプレッドシート：リッチメニュープラス_クラウド保存
```

### 2回目以降の使い方

1. リッチメニュープラスを開く
2. 保存管理のクラウド保存を開く
3. Apps Script URLが入っていることを確認する
4. 「クラウド保存一覧を更新」を押す
5. 読み込みたいクライアント・保存データを選ぶ
6. 「クラウドから読み込み」を押す

### Apps Script URLが分からなくなった時

1. [Google Apps Script](https://script.google.com/) を開く
2. 一覧から、クラウド保存用に作成したプロジェクトを開く
   - 名前を変更していない場合は「無題のプロジェクト」になっていることがあります
3. 右上の「デプロイ」→「デプロイを管理」を押す
4. 種類が「ウェブアプリ」になっているデプロイを開く
5. 表示されている「ウェブアプリURL」をコピーする
   - 末尾は通常 `/exec` です
6. リッチメニュープラスに戻り、「Apps Script URL」欄へ貼り付ける
7. 「接続テスト」を押して、接続OKになるか確認する

### デプロイを管理にURLが表示されない時

- まだWebアプリとしてデプロイできていない可能性があります。
- 「デプロイ」→「新しいデプロイ」から、種類を「ウェブアプリ」にして再度デプロイしてください。
- スクリプトのコードを修正した後は、「デプロイを管理」→鉛筆アイコン→バージョンを「新バージョン」にして保存すると、最新コードが反映されます。

### クラウドに保存する時

1. クライアント名を入力する
2. 保存ファイル名を入力する
3. 必要に応じてチャネルアクセストークンを入力・選択する
4. リッチメニュー画像・エリア・アクションを設定する
5. 「クラウドに保存」を押す

同じクライアント名・保存ファイル名で保存すると、既存のクラウド保存データを更新します。

### 接続できない時の確認

- Apps Script URLが `/exec` で終わっているか確認してください。`/dev` のURLではなく、デプロイ後のウェブアプリURLを使います。
- デプロイ設定の「次のユーザーとして実行」が「自分」になっているか確認してください。
- デプロイ設定の「アクセスできるユーザー」が「全員」になっているか確認してください。
- Apps Scriptのコードを修正した場合は、「デプロイを管理」から新しいバージョンとして再デプロイしてください。
- 保存先のスプレッドシートやDriveフォルダを削除した場合は、接続テストを押すと再作成されます。

## 画像の扱い

クラウド保存は、リッチメニュー画像も含めて保存します。  
そのため、別PCや別ブラウザで開いた場合でも、クラウドから読み込むことで画像・エリア・アクション設定を復元できます。


## 保存先スプレッドシートを開く

保存管理（クライアント管理）を開き、「クラウド保存（Apps Script・Googleスプレッドシート）」を開き、上部の「📊 保存先スプレッドシートを開く」を押すと、Apps Scriptで作成された保存先スプレッドシートを開けます。

先にApps Script URLを設定しておく必要があります。URLが入っていない場合は、「使い方」内のApps Script URL再取得手順に沿ってURLを取得し、Apps Script URL欄に貼り付けてください。

## ブラウザ保存削除

保存管理（クライアント管理）のクライアントカードにある「ブラウザ保存削除」を押すと、そのクライアントのブラウザ内保存だけを削除できます。

削除前に「はい、削除する / いいえ」の確認が表示されます。削除しても、クラウド保存、Google Drive、LINE公式側に反映済みのリッチメニューは削除されません。

クラウド保存欄を開くと、Apps Script URL欄のすぐ上に **📊 保存先スプレッドシートを開く** ボタンが表示されます。

## 今回の表示・動作調整

- クラウド保存欄を初期状態で開くようにし、上部に「📊 保存先スプレッドシートを開く」ボタンを常時表示しました。Apps Script URLから保存先スプレッドシートを取得して開けます。
- 保存管理（クライアント管理）のカードに「ブラウザ保存削除」を追加しました。削除前に「はい、削除する / いいえ」の確認を表示します。
- クラウド保存用スプレッドシートの項目名を日本語化しました。
- 使い方に、Apps Script URLが分からなくなった場合の再取得手順を追加しました。
- 基本設定は展開式ではなく、常時表示に固定しています。
- 保存管理内の「お手軽保存」は、ブラウザ内への保存専用です。LINEには反映しません。
- 「LINEに反映」ボタンで、現在のリッチメニューをLINEへアップロードし、必要に応じて全員表示に設定します。
- LINEに反映後に確認する場合は、LINEアプリを一度終了してから確認してください。反映に数分かかる場合があります。
- それでも反映されない場合は、「LINE表示を停止」でリッチメニューが消えるか確認し、再度「LINEに反映」を押してください。
- 「トーク画面を開いた時にメニューを最初から表示する」は、LINEの `selected` 設定です。複数メニューにチェックが入っていても、同時に複数枚が表示されるわけではありません。実際にユーザーへ表示中のリッチメニューだけに効きます。
- エイリアスIDはリッチメニュー切替で使う目印です。後から変更した場合は、切替元エリアの設定も確認してください。

## Vercel設定

マスター版では、基本的に環境変数は不要です。

チャネルアクセストークンは、画面側の「チャネルアクセストークン」に入力して使えます。  
Vercel側に固定したい場合だけ、以下を設定してください。

```txt
LINE_CHANNEL_ACCESS_TOKEN=チャネルアクセストークン
```

ただし、マスター版はパスワードなしなので、Vercel側にトークンを固定する場合はURL管理に注意してください。

## 昨日のお手軽保存が見えない場合

クラウド保存を追加しても、以前のお手軽保存データは自動ではクラウドに移りません。
同じVercel URLで使っていた場合は、アプリ内の「☁ クラウド保存」内にある「🛟 昨日のお手軽保存を復旧・クラウド移行」から、ブラウザ保存を開くかクラウドへ移行できます。

別のVercel URLや別ブラウザで開いている場合は、ブラウザ保存は別扱いになります。昨日使っていたURLを開き、プロジェクトファイル保存を行ってから、現在のアプリで読み込んでください。
