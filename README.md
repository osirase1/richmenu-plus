# リッチメニュープラス Vercel版

LINE公式アカウントのリッチメニューをブラウザ上で作成し、Vercel API経由でLINEへアップロードするツールです。

## 今回の版について

この版は、シンプル運用向けです。

- Vercel API経由でLINEへアップロード
- デフォルトリッチメニュー設定
- ユーザー単位リンク / リンク解除
- アプリ内パスワードログイン
- API実行時のパスワードチェック
- チャネルアクセストークンをVercelに登録しなくても、画面入力で利用可能
- 現在のデフォルトrichMenuId確認
- デフォルトリッチメニュー解除
- 指定ユーザーに個別リンク中のrichMenuId確認

以下の機能は外しています。

- JSONコード編集タブ
- JSONコピー / 保存 / GUI反映
- エリアテンプレート

## Vercelに必須で登録する環境変数

Vercelのプロジェクト画面で、以下を登録してください。

```txt
APP_PASSWORD=このツールを使うためのログインパスワード
```

登録場所：

```txt
Vercel
↓
対象プロジェクト
↓
Settings
↓
Environment Variables
```

Environmentは、まず `Production` にチェックを入れてください。
Previewにもチェックできる場合は入れてOKです。
Developmentはチェックできなくても問題ありません。

環境変数を保存したら、必ず再デプロイしてください。

```txt
Deployments
↓
最新デプロイの「…」
↓
Redeploy
```

## LINE_CHANNEL_ACCESS_TOKEN について

この版では、`LINE_CHANNEL_ACCESS_TOKEN` をVercelのValueに入れなくても使えます。

APIタブにある「チャネルアクセストークン」欄へ、LINE Developersで発行した長期チャネルアクセストークンを入力してください。

- 「このブラウザにトークンを保存する」にチェックしない場合：ブラウザを閉じると消えます。
- チェックした場合：そのブラウザのローカル保存に残ります。

任意で、Vercelの環境変数に `LINE_CHANNEL_ACCESS_TOKEN` を登録することもできます。その場合は画面入力なしでも動きます。

## パスワード機能について

Vercel公式のPassword Protectionではなく、このツール内に簡易ログイン機能を追加しています。
そのため、追加料金なしで使えます。

ログイン後、ブラウザはAPI通信時に `X-App-Password` ヘッダーを送ります。
Vercel API側では、環境変数 `APP_PASSWORD` と一致する場合のみLINE API操作を許可します。

注意：これは個人用・ポートフォリオ用の簡易保護です。URLを公開しすぎず、パスワードも外部に共有しないでください。


## 共有利用時の仕様・注意書き

このツールは、URLとパスワードを知っている人だけが開ける簡易保護付きツールです。

### 使う人に必要なもの

使う人はVercelアカウントやGitHubアカウントを持っていなくても利用できます。
必要なのは以下の3つです。

```txt
1. このツールのURL
2. APP_PASSWORDで設定したログインパスワード
3. その人が操作したいLINE公式アカウントのチャネルアクセストークン
```

### チャネルアクセストークンの扱い

この版では、`LINE_CHANNEL_ACCESS_TOKEN` をVercelのValueに入れなくても使えます。
使う人がAPIタブにトークンを入力し、アップロード時だけVercel APIへ送信します。

共有相手に使わせる場合は、以下を注意書きとして案内してください。

```txt
・トークンはLINE公式アカウントを操作できる重要情報です。
・このツールは、URLとパスワードを知っている関係者だけが使う前提です。
・共有利用の場合、「このブラウザにトークンを保存する」は基本OFFにしてください。
・利用後はAPIタブの「トークン削除」を押してください。
・公共端末、他人の端末、共有PCではトークンを保存しないでください。
・不特定多数にURLやパスワードを公開しないでください。
・トークンを知られた可能性がある場合は、LINE Developersで再発行してください。
```

### 重要な仕様

- ログインパスワードは `APP_PASSWORD` で管理します。
- LINEトークンは、画面入力または任意の `LINE_CHANNEL_ACCESS_TOKEN` で利用できます。
- 画面入力したトークンは、LINE API操作のたびにVercel APIへ送られます。
- 保存チェックを入れない場合、トークンはブラウザのセッション保存のみです。
- 保存チェックを入れた場合、そのブラウザのローカル保存に残ります。
- この保護は個人用・関係者用の簡易ログインです。会員管理や商用SaaSのような厳密な認証ではありません。

## 使い方

1. Vercelの公開URLを開く
2. `APP_PASSWORD` に設定したパスワードでログイン
3. APIタブでチャネルアクセストークンを入力
4. 画像をアップロード
5. ドラッグでエリアを作成
6. エリアごとにアクションを設定
7. APIタブで「アップロード / 上書き」
8. 全員に表示する場合は「デフォルトに設定」
9. 反映状態確認で「現在のデフォルトrichMenuId」や「個別リンク中richMenuId」を確認

## よくあるエラー

### APP_PASSWORD が未設定です

Vercelの環境変数に `APP_PASSWORD` を追加し、再デプロイしてください。

### チャネルアクセストークンがありません

APIタブにチャネルアクセストークンを入力してください。
または、任意でVercelの環境変数 `LINE_CHANNEL_ACCESS_TOKEN` に登録してください。

### パスワードが違います

ログイン画面で入力したパスワードと、Vercelの `APP_PASSWORD` が一致していません。

### LINE側に表示されない

リッチメニューを作成しただけでは表示されません。
APIタブで「このメニューをデフォルトに設定」を押すか、アップロード時に「アップロード後に自動でデフォルトに設定する」にチェックしてください。

また、過去にユーザー単位リンクを設定している場合、デフォルトよりユーザー単位リンクが優先されます。


## 反映状態確認機能について

APIタブに以下の確認・操作を追加しています。

```txt
・現在のデフォルトrichMenuId確認
・デフォルト解除
・確認用ユーザーIDに個別リンク中のrichMenuId確認
・デフォルト＋個別リンクをまとめて確認
```

LINEでは、ユーザー単位リッチメニューがデフォルトリッチメニューより優先されます。
そのため、デフォルトを更新しても表示が変わらない場合は、確認用ユーザーIDを入力して「個別リンク中のrichMenuId」を確認してください。

注意：デフォルト解除をしても、ユーザー個別リンクが残っている場合、そのユーザーには個別リンク側のリッチメニューが表示されます。


## チャネルアクセストークンを毎回入力しない方法

この版は、以下の2つの方法に対応しています。

### 方法A：このブラウザに保存する

APIタブでチャネルアクセストークンを入力し、`このブラウザにトークンを保存する` にチェックを入れて `トークン保存` を押します。
次回から同じブラウザでは自動入力されます。共有PCでは使わないでください。

### 方法B：Vercel環境変数に保存する

Vercelの `Settings` → `Environment Variables` に以下を追加します。

```text
LINE_CHANNEL_ACCESS_TOKEN=あなたのLINEチャネルアクセストークン
```

この場合、APIタブのトークン入力欄は空欄のまま操作できます。
ただし、URLとAPP_PASSWORDを知っている人は、そのLINE公式アカウントを操作できる状態になります。自分専用・信頼できる関係者用でのみ使ってください。

## デフォルト解除・反映状態確認

APIタブから以下ができます。

- 現在のデフォルトrichMenuId確認
- 自分に個別リンク中のrichMenuId確認
- デフォルト解除

表示が更新されない場合は、個別リンク中のrichMenuIdが古く残っていないか確認してください。

## Google Drive保存機能について

この版では、作成したリッチメニューの編集データを、使う人それぞれのGoogle Driveに保存できます。

保存される内容：

```txt
・メニュー名
・エイリアスID
・チャットバーテキスト
・サイズ
・背景画像
・タップエリア
・アクション設定
・ユーザー情報
・アップロード済みrichMenuId
```

使う人側に必要なのはGoogleアカウントだけです。VercelアカウントやGitHubアカウントは不要です。

### Google Drive保存を使うために必要な環境変数

Vercelに以下を追加してください。

```txt
GOOGLE_CLIENT_ID=Google Cloudで作成したOAuth 2.0 クライアントID
```

すでに必要な `APP_PASSWORD` と合わせると、基本はこうなります。

```txt
APP_PASSWORD=このツールのログインパスワード
GOOGLE_CLIENT_ID=Google OAuth クライアントID
```

`LINE_CHANNEL_ACCESS_TOKEN` は任意です。

### Google Cloud側で必要な設定

Google Drive保存機能を使うには、アプリ管理者がGoogle CloudでOAuthクライアントを作成します。

```txt
1. Google Cloud Consoleを開く
2. プロジェクトを作成
3. Google Drive APIを有効化
4. OAuth同意画面を設定
5. OAuth 2.0 クライアントIDを作成
6. アプリケーションの種類は「ウェブアプリケーション」
7. 承認済み JavaScript 生成元にVercelの公開URLを登録
   例：https://richmenu-plus-vercel.vercel.app
8. 作成されたクライアントIDをVercelの GOOGLE_CLIENT_ID に登録
9. Vercelで再デプロイ
```

OAuth同意画面が「テスト」の場合、使えるGoogleアカウントはテストユーザーに追加したアカウントだけです。複数人に使わせる場合は、テストユーザーに追加するか、Google側の公開設定を調整してください。

### Google Driveへの保存場所

ユーザーがGoogle Drive接続を許可すると、そのユーザーのGoogle Drive内に以下のフォルダを作成します。

```txt
リッチメニュープラス
```

その中に、次のようなJSONファイルとして保存します。

```txt
richmenu-project-メニュー名.json
```

これはLINE APIに送るJSONではなく、このアプリで再編集するためのプロジェクト保存データです。

### 使い方

```txt
1. ツールにログイン
2. Driveタブを開く
3. Google Driveに接続
4. Googleアカウントで許可
5. プロジェクト名を入力
6. Driveに保存
7. 次回はDriveタブから保存済みプロジェクトを開く
```

### 注意

- Google Driveへの接続は、使う人それぞれのGoogleアカウントで許可します。
- 保存データには背景画像も含まれるため、画像が大きいと保存データも大きくなります。
- Google Drive接続のアクセストークンはブラウザのメモリ上で使い、ツールを閉じると再接続が必要になる場合があります。
- Google Driveに保存したデータは、そのユーザーのGoogle Driveに残ります。
